## The Problem

Over years of using a smartphone, I had accumulated thousands of photos stored across a cloud service. When I decided to take back control of my data and download everything locally, I ran into a frustrating problem:

**Downloading photos loses all the good stuff.**

- 📅 Timestamps get stripped or become file-modified dates (wrong)
- 🗺️ GPS location metadata vanishes
- 👤 Face recognition tags don't transfer
- 📂 Files end up as a flat, unsorted dump of thousands of JPGs

Standard file managers are not built for this. I needed something better — but I also needed it to stay **100% on my own machine**, with zero cloud dependency.

---

## The Solution

**LocalSnap** is a self-hosted, browser-accessible photo gallery that runs entirely on your local computer. It is accessed through a web browser pointed at `http://127.0.0.1:5000` — nothing is ever sent to any external server.

### Core Features

| Feature | Description |
|---------|-------------|
| 🖼️ **Timeline View** | Photos grouped and sorted by date taken |
| 🗺️ **Map View** | Interactive world map of where photos were taken |
| 👤 **People & Faces** | Offline face recognition to group photos by person |
| 📁 **Collections** | Auto-grouped: Screenshots, Videos, Places |
| 🔍 **Search** | Filter by date, person, or collection |
| 🔒 **Local-only** | Zero cloud, zero external API calls |

## How Face Recognition Works (Offline)

The face tagging flow is 100% local:

1. **User opens a photo** and types a person's name
2. **YuNet** detects all faces in the photo and draws clickable bounding boxes
3. **User clicks** the correct face
4. **SFace** extracts a 128-dimensional geometric embedding of that face
5. **Background scanner** iterates through the entire library, comparing the stored embedding to every other photo using cosine similarity
6. **Matches are tagged** by writing a `local_people` key into the JSON sidecar file
7. **Gallery rebuilds** — the person now appears in the People view with all their photos

No image is ever sent anywhere. The math happens on your local processor.

## Metadata Recovery

One of the core problems this project solves is recovering metadata from a photo archive export (e.g. Google Takeout). The export includes JSON sidecar files alongside each photo. A Python script reads these sidecars and:

- Restores `DateTimeOriginal` EXIF timestamps using ExifTool
- Embeds GPS coordinates into the photo's EXIF data
- Builds a structured `gallery_data.js` index for the frontend

```python
# Snippet: reading timestamps from a JSON sidecar
import json
from datetime import datetime

with open("photo.jpg.supplemental-metadata.json") as f:
    meta = json.load(f)

unix_ts = int(meta["photoTakenTime"]["timestamp"])
date_taken = datetime.fromtimestamp(unix_ts)
# → 2023-07-14 18:42:00
```

## Face Detection Snippet

```python
# Multi-scale face detection using OpenCV YuNet
# Runs entirely on local CPU — no internet required

import cv2

detector = cv2.FaceDetectorYN.create(
    "models/face_detection_yunet.onnx",
    "", (320, 320),
    score_threshold=0.5,   # Permissive — catches angled/partial faces
    nms_threshold=0.3,
    top_k=5000
)

def detect_faces(image_path):
    img = cv2.imread(image_path)
    h, w = img.shape[:2]
    detector.setInputSize((w, h))
    _, faces = detector.detect(img)
    return faces  # Returns bounding boxes + landmarks
```

## Live Scan Progress

Rather than waiting blindly, the UI shows a live progress bar during the library scan:

```
┌─────────────────────────────────────────────────┐
│ ● Scanning for [Person]...                  67%  │
│ ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓░░░░░░░░░              │
│ 9,842 / 14,600 photos   ✅ 38 matches found     │
└─────────────────────────────────────────────────┘
```

The scan status is polled every 2 seconds from a `/api/scan_status` endpoint. When complete, the gallery auto-refreshes and the People view populates.
